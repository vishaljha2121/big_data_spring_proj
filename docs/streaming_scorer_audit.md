# Streaming Scorer Audit

## Status

Milestone 3B status: **PASSED** for the local JSONL scoring path.

## Input Event Source

The scorer consumes canonical replay point events from:

```text
data/results/replay_dry_run/sample_events.jsonl
```

Those events are generated by `producer/replay_producer.py` from:

```text
data/replay/manifests/replay_manifest_v1.parquet
```

The input contract remains `contracts/point_event_schema.json`. The scoring path does not read raw CSV, staging CSV.GZ, or generated model feature Parquet.

## Model Loading Path

The runtime loader reads:

```text
data/models/odds/latest.json
data/models/odds/v1/model.joblib
data/models/odds/v1/feature_schema.json
data/models/risk/latest.json
data/models/risk/v1/risk_config.json
```

`streaming/model_loader.py` validates the feature schema hash and rejects inference if any required training feature is missing.

## Online Feature Builder

`streaming/online_feature_builder.py` keeps state per `synthetic_match_id`. For each event it:

1. builds model-ready features from prior state only;
2. scores the current event;
3. updates state with the current event outcome.

This preserves point-in-time safety for the published odds model.

## Scoring Output

The primary output is:

```text
data/results/streaming_scoring/scored_events_sample.jsonl
data/results/streaming_scoring/scored_events_sample.parquet
```

The output contract is `contracts/scored_event_schema.json`.

## Validation Result

The local scoring run processed `1000` events with `0` invalid events. Validation passed with:

```text
data/results/streaming_scoring/scoring_validation_report.json
```

## Limitations

- The probability fields are point-level probabilities, not match-win probabilities and not betting odds.
- Kafka scoring runner code exists, but Kafka runtime was not executed in this environment.
- The risk score is a conservative statistical anomaly signal only. It is not match-fixing detection and is not proof of misconduct.
